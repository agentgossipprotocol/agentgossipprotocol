# Governance

This document describes how decisions are made for the Agent Gossip Protocol.

## Roles

- **Maintainers** — review and merge changes, steward releases, and have final
  say on SEP acceptance.
- **Contributors** — anyone who opens issues, pull requests, or SEPs.

## Decision-making

- Editorial changes (typos, clarifications) may be merged by any maintainer.
- Substantive changes require a **Specification Enhancement Proposal (SEP)**
  (see [`proposals/`](proposals/)) and maintainer consensus.
- Where consensus cannot be reached, maintainers decide by simple majority.

## Releases and versioning

The specification uses [Semantic Versioning](https://semver.org/):

- **MAJOR** — backwards-incompatible protocol or schema changes.
- **MINOR** — backwards-compatible additions.
- **PATCH** — editorial or non-normative clarifications.

Each release is tagged and recorded in [`CHANGELOG.md`](CHANGELOG.md).

Changes to this document follow the SEP process.
